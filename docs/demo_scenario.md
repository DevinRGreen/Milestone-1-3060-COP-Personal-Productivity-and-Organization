# Demo Scenario

### Step 1 — User Logs In
- A user opens the app.
- They log in using their registered email and password.
- The dashboard loads showing current tasks.

### Step 2 — User Creates a Task
- User clicks “Create Task.”
- They enter:
  - Task name
  - Category (school, work, personal)
  - Priority level
  - Due date
- The system saves the task to the database.
- The new task appears on the dashboard.

### Step 3 — User Edits a Task
- User selects an existing task.
- They update the deadline or priority.
- System updates the record and dashboard refreshes.

### Step 4 — User Deletes a Task
- User selects a completed task.
- They click “Delete.”
- The system removes it from the database.

### Step 5 — System Shows Organized Results
- Dashboard updates instantly:
  - Tasks are sorted by due date.
  - High-priority tasks appear clearly.
  - Upcoming deadlines section updates.

### Step 6 — API Interaction (if implemented in demo)
- The system syncs a task’s due date with an external time/calendar API.
- User sees confirmation or an updated reminder time.

This flow demonstrates the core value and functionality of the MVP.
