# MVP Demo Requirements

To successfully run the live system demonstration, the MVP must include:

## User Features
- User registration and login
- Ability to create, edit, and delete tasks
- Ability to assign:
  - Category
  - Priority
  - Due date
- Dashboard to display all tasks and upcoming deadlines

## System Features
- Database to store user tasks
- CRUD operations for task management
- Secure session handling
- Automatic refresh or visible update of the task list

## API Integration (Task 5)
- Ability to send task due dates to external time/calendar API
- Use API data to support reminders or time formatting

## Demo Requirements
- Demo should run in ~3 minutes
- Only one new task will be created live during demo
- All other data can be pre-seeded to show dashboard functionality
